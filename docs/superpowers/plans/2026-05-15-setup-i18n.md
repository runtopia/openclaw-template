# Setup Wizard i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add Chinese/English bilingual support to `setup.html` with automatic browser language detection.

**Architecture:** Inline JS translation dictionary (`I18N`) + Alpine.js reactive binding. A `t(key)` method on the Alpine data object looks up translations based on `navigator.language`. Zero external dependencies, no build step.

**Tech Stack:** Vanilla JS, Alpine.js (already in use), Tailwind CSS (already in use)

- [ ] **Step 2: Add `lang` property and `t()` method to Alpine data object**

In the `Alpine.data('setupApp', () => ({` object, add:

```javascript
  lang: 'en',

  detectLanguage() {
    const navLang = (navigator.language || navigator.userLanguage || 'en').toLowerCase();
    this.lang = navLang.startsWith('zh') ? 'zh' : 'en';
    document.documentElement.lang = this.lang;
  },

  t(key, ...args) {
    let text = (I18N[this.lang] && I18N[this.lang][key]) || I18N['en'][key] || key;
    if (args.length > 0) {
      args.forEach((arg, i) => {
        text = text.replace(`{${i}}`, arg);
      });
    }
    return text;
  },
```

- [ ] **Step 3: Call `detectLanguage()` in `init()`**

In the `init()` method, add `this.detectLanguage();` as the first line:

```javascript
async init() {
  this.detectLanguage();
  await this.refreshStatus();
},
```
